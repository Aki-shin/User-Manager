# User Manager

Веб-приложение для управления учётными записями пользователей в **FreeIPA** с синхронизацией из XLSX-файлов кадровой системы и интеграцией с **GLPI**.

## Возможности

- **Управление пользователями FreeIPA** — просмотр, создание, редактирование, удаление, сброс пароля, блокировка от синхронизации, поиск.
- **Синхронизация из XLSX** — загрузка выгрузки из кадровой системы и автоматическое сравнение с FreeIPA. Приложение показывает разницу (новые / изменённые / удалённые пользователи) и применяет выбранные изменения пакетами с прогресс-баром.
- **Рассылка учётных данных** — отправка логина и пароля сотруднику на email. Письма идут через очередь с настраиваемой задержкой, чтобы при массовом создании учёток не сработали антиспам-фильтры почтового сервера.
- **Интеграция с GLPI** — после применения изменений приложение по SSH запускает встроенную команду GLPI `glpi:ldap:synchronize_users`, и GLPI обновляет свою базу пользователей из FreeIPA.
- Веб-интерфейс на русском языке (Bootstrap 5).

## Стек

- Python 3, Flask
- `openpyxl` — разбор XLSX
- `requests` — JSON-RPC клиент к FreeIPA
- `paramiko` — SSH для запуска LDAP-синхронизации GLPI
- Bootstrap 5 — фронтенд
- Целевое окружение: Debian 12, FreeIPA 4.x

## Структура

```
app.py            Flask-маршруты и бизнес-логика
config.py         Конфигурация и хранилище паролей
ipa_client.py     JSON-RPC клиент FreeIPA

```

## Установка

На сервере Debian 12 от имени root:

```bash
git clone https://github.com/Aki-shin/User-Manager.git
cd User-Manager
sudo ./deploy.sh
```

Скрипт установит зависимости, создаст systemd-сервис `user-manager` и запустит приложение на порту `5000`.

Управление сервисом:

```bash
systemctl {start|stop|restart|status} user-manager
journalctl -u user-manager -f
```

### Запуск для разработки

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python app.py
```


Адрес и порт переопределяются переменными окружения `HOST` и `PORT` (по умолчанию `127.0.0.1:5000`).

## Настройка

Все параметры задаются через веб-интерфейс в разделе **Настройки**:

- **FreeIPA** — адрес сервера, учётная запись администратора, целевая группа пользователей.
- **Почта (SMTP)** — сервер, порт, шифрование, шаблоны писем, задержка между отправками.
- **GLPI** — SSH-доступ к серверу GLPI, ID LDAP-сервера, команда синхронизации.

### Интеграция с GLPI

Приложение подключается к серверу GLPI по SSH и запускает CLI-команду:

```bash
sudo -u www-data php /var/www/html/glpi/bin/console \
    glpi:ldap:synchronize_users --ldap-server-id=1 --no-interaction
```

Чтобы SSH-пользователь мог выполнять её без ввода пароля, на стороне GLPI
добавьте правило в `/etc/sudoers.d/glpi-sync`:

```
<ssh-user> ALL=(www-data) NOPASSWD: /var/www/html/glpi/bin/console
```

Для аутентификации рекомендуется SSH-ключ: путь к приватному ключу указывается
в настройках, публичный ключ добавляется в `authorized_keys` на сервере GLPI.

## Файлы данных

Создаются автоматически в рабочем каталоге и **не попадают в репозиторий**
(перечислены в `.gitignore`):

- `config.json` — настройки приложения
- `passwords.json` — сохранённые пароли пользователей (для повторной отправки)
- `locks.json` — пользователи, заблокированные от синхронизации
- `mail_queue.json` — очередь писем

`deploy.sh` сохраняет эти файлы при переустановке.

## Безопасность

Приложение рассчитано на работу во внутренней сети. Пароли (FreeIPA, SMTP, SSH,
а также пароли пользователей) хранятся в JSON-файлах в открытом виде — каталог
приложения должен быть доступен только сервисной учётной записи. Не выставляйте
сервис в интернет без обратного прокси с HTTPS и ограничением доступа.
